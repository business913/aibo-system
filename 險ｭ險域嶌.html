1. システム概要

AIBOは、LINE・Web・アプリを統合したAI不動産ブローカー管理プラットフォームです。物件情報の受付から商流確認・報酬計算・契約完了まで、不動産取引のすべてをデジタルで完結させます。

システム名: AIBO（AI不動産相棒）

  • LINE・PCどちらからでも物件情報を登録・閲覧できる
  • AIが概要書を読み取り、自動で物件データを構造化
  • 商流をリレー式に確認し、売主まで追跡する
  • 報酬を商流人数・プランに応じてリアルタイムに自動計算
  • Google Cloud / Firebase による世界展開対応

3. LINE Webhook 実装詳細

⚠️ MANUSで断念した問題の解決方法

LINEのWebhookはインターネット公開済みのHTTPS URLが必須です。Google Cloud Runはデプロイと同時にHTTPS URLが自動発行されるため、この問題を完全に解決します。

  • HTTPS自動付与（LINEの要件を満たす）
  • LINE署名検証（x-line-signature）を実装済み
  • LINEへの200レスポンスを即時返却（タイムアウト防止）
  • イベント処理は非同期で実行

署名検証フロー

  LINE Platform
      │
      │ POST /webhook
      │ Header: x-line-signature: [Base64(HMAC-SHA256(body, Channel Secret))]
      │ Body: {"events": [...]}
      ▼
  server.js
      │
      ├─ req.rawBody を保持（express.json の verify オプション）
      │
      ├─ HMAC-SHA256(rawBody, LINE_CHANNEL_SECRET) を計算
      │
      ├─ crypto.timingSafeEqual() でタイミング攻撃を防いで比較
      │
      ├─ 一致 → 200 OK を即返却 → イベント非同期処理
      └─ 不一致 → 401 Unauthorized

メッセージ処理フロー（状態マシン）

ユーザー状態 (state)受信内容処理
default「物件登録」または「1」状態をawaiting_property_textに変更
awaiting_property_textテキストメッセージGPT-4oで解析 → Firestoreに保存 → 商流確認開始
awaiting_property_text画像LINE Content APIで取得 → OCR → GPT-4o解析
awaiting_request_textテキストメッセージ希望案件として保存 → 全会員にブロードキャスト
defaultその他メインメニューFlexメッセージを表示

4. 商流確認システム

商流確認アルゴリズム

  1. AさんがLINEで物件情報を送信

  2. サーバーが物件をFirestoreに保存し、商流コード（例: AB34XK）を発行

  3. Aさんにメッセージ:
     「あなたに情報をくれた方にこのコードを共有してください: AB34XK」

  4. BさんがLINEで「AB34XK」を入力

  5. サーバーがBさんにメッセージ:
     「この物件情報をAさんに渡しましたか？」
     → YES の場合 → Bさんを商流チェーンに追加
     → 新しいコードを発行してBさんの紹介元へ繰り返す

  6. 売主に到達したら:
     → 本人確認書類をLINEで受領
     → 商流確認完了フラグをFirestoreに保存
     → AIBOから売主に挨拶メッセージを送信
     → 管理者に通知

  Firestore 保存形式:
  flowVerifications/{propertyId}
  {
    chain: [
      { userId: "U111", role: "broker",  verifiedAt: Timestamp },
      { userId: "U222", role: "agent",   verifiedAt: Timestamp },
      { userId: "U333", role: "seller",  verifiedAt: Timestamp }
    ],
    status: "reached_seller"
  }

5. 報酬計算エンジン

計算ロジック（売主側）

物件価格 × 1.5%
= 売主側手数料

  うち 0.5% = 紹介者プール
  うち 1.0% = AIBO取り分

紹介者プール ÷ 紹介者数
= 各紹介者の報酬

PRO会員の場合: +0.1%加算
月5件以上の場合: さらに+0.1%加算

計算例（¥3.2億の物件・紹介者2名）

物件価格: ¥320,000,000
売主手数料(1.5%): ¥4,800,000

  紹介者プール(0.5%): ¥1,600,000
    紹介者A(FREE): ¥800,000
    紹介者B(PRO): ¥800,000
                  + ¥320,000
                  = ¥1,120,000

  AIBO取り分(1.0%): ¥3,200,000

7. セキュリティ設計

実装済みのセキュリティ

  • LINE署名検証（HMAC-SHA256）
  • タイミング攻撃対策（timingSafeEqual）
  • APIキーはSecret Managerで管理
  • Dockerコンテナは非rootユーザーで実行
  • Firestoreセキュリティルールで認証済みユーザーのみアクセス
  • HTTPS通信のみ（Cloud Run自動対応）

個人情報の取り扱い

  • 身分証明書は暗号化してCloud Storageに保存
  • 口座情報は別途暗号化DB（AES-256）
  • 商流図は名前を表示せず役割のみ
  • GDPR・個人情報保護法に準拠した設計
  • 管理者のみがPII（個人情報）にアクセス可能
